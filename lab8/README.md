# Лабораторна робота №8

## Тема

Огляд сучасного популярного автоматизованого інструментарію.

## Варіант №1

Cypress & TestCafe.

## Мета роботи

Метою роботи є коротке дослідження двох популярних JavaScript-інструментів для автоматизованого UI-тестування, порівняння їхніх можливостей і реалізація однакових перевірок для демонстраційного веб-додатку SauceDemo.

## Обраний ресурс для тестування

Для тестування використано ресурс [SauceDemo](https://www.saucedemo.com).

Дані для входу:

- username: `standard_user`
- password: `secret_sauce`

SauceDemo є демонстраційним веб-додатком інтернет-магазину з реальною login-формою, товарами та сторінкою Products. Це дозволяє порівняти Cypress і TestCafe на однакових UI-перевірках реального користувацького інтерфейсу.

## Коротке дослідження Cypress

Cypress - це сучасний JavaScript-фреймворк для end-to-end, інтеграційного та компонентного тестування веб-додатків. Він запускає тести безпосередньо в браузері, має зручний режим налагодження, автоматичне очікування елементів, зрозумілі повідомлення про помилки, скріншоти та відеозаписи падінь тестів.

Сильні сторони Cypress:

- простий і читабельний синтаксис тестів;
- автоматичні очікування без великої кількості ручних пауз;
- зручний Test Runner для налагодження;
- хороша інтеграція з CI/CD;
- швидке написання UI та E2E-тестів для сучасних frontend-додатків.

Слабкі сторони Cypress:

- обмеження для сценаріїв із кількома вкладками або кількома браузерними контекстами;
- фокус переважно на браузерному тестуванні;
- деякі складні cross-origin сценарії потребують додаткового налаштування;
- для великих наборів тестів може знадобитися оптимізація часу виконання.

Типові задачі Cypress:

- перевірка авторизації користувача;
- тестування форм, кнопок, навігації та таблиць;
- end-to-end перевірка основних користувацьких сценаріїв;
- regression testing після змін у frontend-частині;
- запуск UI-тестів у CI/CD.

Висновок щодо зручності використання: Cypress дуже зручний для швидкого старту та розробки тестів, особливо коли потрібно бачити процес виконання, швидко знаходити причину помилки й підтримувати читабельні перевірки.

## Коротке дослідження TestCafe

TestCafe - це Node.js-інструмент для end-to-end тестування веб-додатків. Він не потребує WebDriver, має власний механізм взаємодії з браузером і дозволяє запускати тести у різних браузерах. TestCafe підтримує асинхронний синтаксис, селектори, автоматичні очікування та запуск тестів у headless-режимі.

Сильні сторони TestCafe:

- не потребує Selenium WebDriver;
- підтримує запуск у різних браузерах;
- має зрозумілий async/await синтаксис;
- автоматично очікує появу елементів;
- добре підходить для кросбраузерних UI-перевірок.

Слабкі сторони TestCafe:

- менша популярність і менша екосистема порівняно з Cypress;
- налагодження може бути менш наочним;
- частина сучасних frontend-сценаріїв може вимагати додаткових налаштувань;
- виконання тестів іноді повільніше через проксі-механізм TestCafe.

Типові задачі TestCafe:

- кросбраузерне end-to-end тестування;
- перевірка login-форм і базової навігації;
- тестування користувацьких сценаріїв у Chrome, Edge, Firefox та інших браузерах;
- запуск UI-тестів без WebDriver;
- перевірка стабільності інтерфейсу після змін.

Висновок щодо зручності використання: TestCafe є зручним варіантом для кросбраузерного тестування без WebDriver. Його синтаксис зрозумілий, але загальна екосистема та інструменти налагодження менш розвинені, ніж у Cypress.

## Реалізовані UI-тести

Для Cypress реалізовано 3 тести:

1. перевірка відкриття login-сторінки SauceDemo;
2. перевірка наявності полів username, password і кнопки Login;
3. успішна авторизація користувача `standard_user` / `secret_sauce` і перевірка заголовка `Products`.

Для TestCafe реалізовано 3 тести:

1. перевірка відкриття login-сторінки SauceDemo;
2. перевірка наявності полів username, password і кнопки Login;
3. успішна авторизація користувача `standard_user` / `secret_sauce` і перевірка заголовка `Products`.

## Команди запуску

Перед першим запуском потрібно встановити залежності:

```bash
npm install
```

Запуск Cypress у headless-режимі:

```bash
npm run test:cypress
```

Запуск TestCafe у headless-режимі Chrome:

```bash
npm run test:testcafe
```

Запуск обох наборів тестів послідовно:

```bash
npm test
```

Якщо в середовищі виконання немає доступного Chrome або Chromium, браузерні тести потрібно запускати локально після встановлення браузера.
